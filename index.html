1. Şəbəkə və onun növləri


Şəbəkə iki və ya daha çox cihazın məlumat mübadiləsi üçün

bir-birinə qoşulmasıdır.Əsas növləri məhdud bir ərazidəki

cihazları birləşdirən LAN (Local Area Network) və daha geniş

coğrafi əraziləri birləşdirən WAN (Wide Area Network) ilə təmsil

olunur. MAN (Metropolitan Area Network) bir şəhər ərazisini

əhatə edir, PAN (Personal Area Network) isə fərdi cihazları

əlaqələndirir. Şəbəkələr, həmçinin, qoşulma topologiyasına

(ulduz, halqa) və idarəetmə modelinə (peer-to-peer, client-server)

görə də fərqlənir. Bu strukturlar məlumatların effektiv və təhlükəsiz

ötürülməsini təmin edir.

2. Şəbəkə avadanlıqları və onların işləmə mexanizmləri


Əsas şəbəkə avadanlıqları məlumatların ötürülməsini,

yönləndirilməsini və idarə edilməsini təmin edir.

Kommutatorlar (Switches) MAC ünvanlarına əsaslanaraq məlumatı

(frame) eyni LAN daxilində dəqiq ünvana çatdırır. Routerlər

isə IP ünvanlarına baxaraq müxtəlif şəbəkələr arasında məlumat

paketlərini yönləndirir və marşrutlama qərarları qəbul edir.

Firewall-lar təhlükəsizlik qaydalarına əsasən daxil olan və

çıxan trafiki filtrələyir. Hub-lar (təkrarlayıcılar) sadəcə

olaraq siqnalı təkrar edir, lakin effektiv deyillər.

3. OSI modeli


OSI (Open Systems Interconnection) modeli şəbəkə rabitə

prosesini standartlaşdıran yeddi təbəqəli konsepsiyadır.

Bu təbəqələr Fizikidən (1-ci təbəqə, bitlər), Məlumat

Bağlantısına (2-ci təbəqə, kadrlar), Şəbəkəyə

(3-cü təbəqə, paketlər), Nəqliyyata (4-cü təbəqə, seqmentlər),

Seansa, Təqdimata və Tətbiqə (7-ci təbəqə) qədər məlumatın

hərəkətini təsvir edir. Model müxtəlif istehsalçıların

avadanlıqlarının bir-birilə əlaqə qurmasına imkan verir.

Hər təbəqə yalnız yuxarı və aşağı təbəqələrlə qarşılıqlı

əlaqədə olur.

4. Nəqliyyat təbəqəsi və protokolları


Nəqliyyat təbəqəsi (OSI-də 4-cü təbəqə) məlumatların

son nöqtələr arasında etibarlı və nizamlı ötürülməsindən

məsuldur. Əsas protokolları TCP (Transmission Control Protocol)

və UDP (User Datagram Protocol) təşkil edir. TCP əlaqəyə

yönəlik (connection-oriented) olub, məlumatın çatdırılmasına

zəmanət verir və itirilmiş paketləri bərpa edir. UDP isə

əlaqəsiz (connectionless) və sürətli ötürmə üçün istifadə

olunur, lakin çatdırılmaya zəmanət vermir. Bu təbəqə məlumatları

seqmentlərə bölür və yenidən yığır, həmçinin port

nömrələri ilə tətbiqləri fərqləndirir.

5. Şəbəkə təbəqəsi, protokolları və əsas göstəriciləri


Şəbəkə təbəqəsi (OSI-də 3-cü təbəqə) şəbəkələr arasında

məlumat paketlərinin mənbədən təyinat nöqtəsinə çatdırılmasına

cavabdehdir. Əsas protokolu IP (İnternet Protokolu) olub, hər

cihaza unikal ünvan (IP ünvan) təyin edir. Bu təbəqə marşrutlaşdırma

qərarlarını qəbul edir. Məlumatlar bu təbəqədə paket adlanır.

Əsas göstəricilər arasında marşrut seçimi, ünvanlama və tıxacın

idarə edilməsi yer alır.

6. Router, marşrutlama cədvəli və routerin əsas xüsusiyyətləri


Routerlər (Marşrutlaşdırıcılar) fərqli IP şəbəkələri arasında

paketləri yönləndirən şəbəkə cihazlarıdır. Onlar paketlərin

hara göndərilməli olduğuna qərar vermək üçün marşrutlama

cədvəllərindən istifadə edirlər. Bu cədvəllər ya statik

şəkildə administrator tərəfindən daxil edilir, ya da dinamik

marşrutlama protokolları vasitəsilə avtomatik öyrənilir.

Routerlər müxtəlif marşrutları müqayisə etmək üçün

metrikalardan (məsələn, ən qısa yol) istifadə edir. Onlar

şəbəkələri seqmentləşdirərək trafikin idarə edilməsində və

təhlükəsizliyində vacib rol oynayır.

7. Məlumat bağlantısı təbəqəsi və onun əsas göstəriciləri


Məlumat bağlantısı təbəqəsi (OSI-də 2-ci təbəqə) eyni

şəbəkə seqmentində olan cihazlar arasında məlumatların etibarlı

ötürülməsini təmin edir. Bu təbəqədə məlumatlar kadr (frame)

adlanır və unikal MAC ünvanları (Media Access Control) ilə

identifikasiya edilir. Təbəqə iki alt təbəqəyə bölünür: LLC

(Logical Link Control) və MAC. Əsas funksiyalara səhv

aşkarlanması, fiziki ünvanlama və mühitə daxil olma nəzarəti

(məsələn, Ethernet və Wi-Fi) daxildir.

8. Fiziki təbəqə və onun əsas göstəriciləri


Fiziki təbəqə (OSI-də 1-ci təbəqə) şəbəkə mühiti üzərindən

xam rəqəmsal bitlərin fiziki ötürülməsinə cavabdehdir. Bu,

elektrik siqnalları, işıq impulsları və ya radio dalğaları

şəklində həyata keçirilir. Əsas göstəricilərə kabel növləri

(mis, optik lif), birləşdiricilərin standartları (RJ-45),

siqnalın kodlaşdırılması, ötürmə sürəti və fiziki topologiya

daxildir. Bu təbəqədə avadanlıqlara kabellər, təkrarlayıcılar

(repeaters) və hub-lar aid edilir.

9. Məlumatın kapsullaşdırılması və əks proses


Məlumatın kapsullaşdırılması OSI modelində bir tətbiqdən

(7-ci təbəqə) fiziki mühitə (1-ci təbəqə) doğru hərəkət

edərkən hər təbəqənin məlumata öz başlıq və ya quyruğunu

əlavə etmə prosesidir. Məsələn, Nəqliyyat təbəqəsi başlıq

əlavə edərək paketi Seqmentə çevirir, Şəbəkə təbəqəsi IP başlıq

əlavə edərək onu Paketə çevirir və s. Əks proses isə məlumatın

təyinat nöqtəsində tərs ardıcıllıqla hər təbəqədən keçərkən müvafiq

başlıqların çıxarılmasıdır.

10. Nəqliyyat təbəqəsi port nömrələri


Nəqliyyat təbəqəsi port nömrələri bir IP ünvanına çatan məlumatı

hansı tətbiqə və ya xidmətə çatdırmaq lazım olduğunu müəyyən edir.

Port nömrələri 0-dan 65535-ə qədərdir və üç əsas kateqoriyaya

bölünür. Yaxşı Tanınmış Portlar (Well-Known Ports, 0-1023) standart

xidmətlər (məsələn, HTTP üçün 80, HTTPS üçün 443) üçün istifadə olunur.

Qeydə alınmış Portlar (Registered Ports, 1024-49151) müəyyən

tətbiqlər üçün qeyd olunur. Özəl/Dinamik Portlar (49152-65535)

isə müştəri tətbiqləri tərəfindən müvəqqəti istifadə olunur.

11. Cisco iyerarxik modeli


Cisco iyerarxik modeli şəbəkəni üç əsas təbəqəyə bölərək dizayn

etməyi tövsiyə edir: Giriş (Access), Paylama (Distribution)

və Əsas (Core). Giriş təbəqəsi son istifadəçiləri şəbəkəyə birləşdirən

kommutatorları ehtiva edir. Paylama təbəqəsi Giriş təbəqələrindən

gələn trafikin idarə edilməsi, marşrutlaşdırma və siyasətlərin tətbiqi

(məsələn, ACL-lər) üçündür. Əsas təbəqə isə şəbəkədə yüksək

sürətli magistral rabitəni təmin edir və sürətli ötürməyə yönəlmişdir.

Bu model şəbəkənin miqyaslanmasını, təhlükəsizliyini və idarəetməsini

asanlaşdırır.

12. TCP/IP modeli və protokolları


TCP/IP (Transmission Control Protocol/Internet Protocol) modeli

İnternetin əsasını təşkil edən dörd və ya beş təbəqəli bir modeldir

(Tətbiq, Nəqliyyat, İnternet/Şəbəkə, Şəbəkə Girişi). Bu model OSI

modelinə nisbətən daha praktiki bir yanaşmadır. Əsas protokollara

Tətbiq təbəqəsində HTTP, DNS; Nəqliyyat təbəqəsində TCP və UDP;

İnternet təbəqəsində isə IP, ICMP daxildir. TCP/IP modeli şəbəkələrarası

rabitəni təmin edir və bütün müasir şəbəkələrin əsas arxitekturasıdır.

13. IP ünvanlama və növləri


IP ünvanlama cihazları şəbəkədə unikal olaraq tanımaq üçün

istifadə olunan məntiqi ünvanlama sistemidir. Ən geniş yayılmış növlər

32-bitli IPv4 və 128-bitli IPv6-dır. IPv4 ünvanları siniflərə (A, B, C)

bölünür, həmçinin xüsusi ünvanlar (məsələn, şəxsi ünvanlar, loopback)

mövcuddur. IPv6 ünvan çatışmazlığı problemini həll edir və təhlükəsizlik

xüsusiyyətlərini təkmilləşdirir. IP ünvanlar cihazların şəbəkələr

arasında bir-birini tapmasına imkan verir.

14. Alt şəbəkələr və tətbiqi


Alt şəbəkə (Subnetting) böyük bir şəbəkəni daha kiçik, idarəolunan

hissələrə bölmə prosesidir. Bu, IP ünvanlarının səmərəli istifadəsini

artırır və yayım (broadcast) trafikini azaldır. Bütün IP sinifləri

(A, B, C) üçün tətbiq edilir, lakin müasir şəbəkələrdə CIDR

(Classless Inter-Domain Routing) əsasında VLSM (Variable Length Subnet Mask)

istifadəsi üstünlük təşkil edir. Praktiki tətbiq şəbəkə maskasının

dəyişdirilməsi yolu ilə host sayını müəyyən etmək və təhlükəsizliyi

təmin etməkdən ibarətdir.

15. Şəbəkə əməliyyat sistemi (NOS)


NOS (Network Operating System) şəbəkə avadanlıqlarını idarə

edən və şəbəkə resurslarına çıxışı təmin edən proqram

təminatıdır. Əsas funksiyalarına istifadəçi və resurs idarəetməsi,

təhlükəsizlik, habelə marşrutlaşdırma və kommutasiya xidmətləri

daxildir. Növlərinə Microsoft Windows Server, Linux (Red Hat, Ubuntu)

və ya Cisco IOS kimi cihaz OS-ləri aid edilir. NOS-un gələcəyi Cloud

və SDN (Software-Defined Networking) texnologiyalarına inteqrasiya olunmuş,

daha çevik və avtomatlaşdırılmış sistemlərdə görülür.

16. Cisco-nun IOS sistemi


Cisco IOS (Internetwork Operating System) Cisco router və

kommutatorları kimi şəbəkə avadanlıqlarının əsas əməliyyat sistemidir.

IOS cihazların funksionallığını idarə edir, o cümlədən marşrutlaşdırma,

kommutasiya, təhlükəsizlik (ACL-lər) və şəbəkə interfeyslərinin

konfiqurasiyası. O, administratorlara cihazları Əmr Sətri İnterfeysi

(CLI) vasitəsilə idarə etmək imkanı verir. IOS şəbəkə cihazlarının

dayanıqlığını və miqyaslanmasını təmin edən əsas proqram təminatıdır.

17. Cisco IOS cihazlarına qoşulma


Cisco IOS cihazlarına qoşulma prosesi əsasən üç yolla həyata

keçirilir: Konsol portu (ilkin konfiqurasiya üçün fiziki əlaqə),

Telnet/SSH (şəbəkə üzərindən uzaqdan təhlükəsiz və ya təhlükəsiz olmayan əlaqə)

və AUX portu (dial-up modem vasitəsilə ehtiyat əlaqə). Konsol qoşulması

üçün xüsusi konsol kabeli və terminal emulyator proqramı istifadə olunur.

Uzaqdan qoşulmalar təhlükəsizlik üçün adətən SSH (Secure Shell)

vasitəsilə şifrələnərək həyata keçirilir.

18. CLI və promptlar


CLI (Command Line Interface) şəbəkə administratorlarına mətn

əmrləri vasitəsilə şəbəkə cihazlarını konfiqurasiya etməyə

və idarə etməyə imkan verir. Cisco IOS-da daxil olmaq üçün

müxtəlif promptlar mövcuddur: User EXEC Mode (>), Privileged

EXEC Mode (#) və Global Configuration Mode ((config)#). Enable

əmri User rejimindən Privileged rejimə keçməyə imkan verir.

CLI prompbları administratorun hazırda hansı konfiqurasiya

səviyyəsində olduğunu göstərir.

19. ACL-lər və konfiqurasiya


Girişə icazə siyahıları (ACL) şəbəkə cihazlarında (routerlər, kommutatorlar)

trafikin icazə verilmiş və ya rədd edilmiş olmasını müəyyən edən qaydalar

toplusudur. ACL-lər Standart (yalnız mənbə IP ünvanına əsaslanır) və

Genişləndirilmiş (mənbə/təyinat IP, port və protokol növünə əsaslanır)

növlərə ayrılır. Konfiqurasiya edilərkən, qaydalar yuxarıdan aşağıya

icra olunur və sonuncu qayda hər şeyi rədd edən (implicit deny) bir əmrdir.

ACL-lər təhlükəsizliyi təmin etmək və şəbəkə trafikini nəzarətdə

saxlamaq üçün istifadə olunur.

20. Dinamik marşrutlama və protokollar


Marşrutlama protokolları routerlərə şəbəkə marşrutlarını avtomatik

öyrənməyə və ən yaxşı yolu seçməyə kömək edir. Onlar Məsafə Vektoru

(Distance Vector, məsələn, RIP) və Link Vəziyyəti (Link State, məsələn, OSPF)

növlərinə bölünür. OSPF (Open Shortest Path First) Link Vəziyyəti

protokolu olub, hər routerin bütün şəbəkənin xəritəsini (topologiyasını)

yaratmasını təmin edir. Konfiqurasiyası router ospf [process-id] əmri

ilə başlayır və şəbəkə sahələri müəyyən edilərək (Area) ən qısa yolu

tapmaq üçün Dijkstra alqoritmindən istifadə edir.

21. SOC loqlar və PPTGC


SOC (Security Operations Center) təhlükəsizlik hadisələrinin

monitorinqi, analizi və onlara reaksiya verilməsi üçün mərkəzləşdirilmiş

komandadır. Loqların toplanılması bütün şəbəkə cihazlarından və

tətbiqlərdən məlumatların SIEM (Security Information and Event Management)

sisteminə ötürülməsi ilə həyata keçirilir. Əsas komponentlər

İnsanlar (analitiklər), Proseslər (reaksiya planları), Texnologiyalar

(SIEM, SOAR), İdarəetmə (Governance) və Uyğunluq (Compliance)

elementlərindən ibarətdir (PPTGC).

22. SOC arxitekturası


SOC-un qlobal arxitekturası təhlükəsizliyi təmin etmək üçün

müəyyən bir çərçivə təqdim edir. Bu arxitektura log toplama və

saxlama sistemlərini, təhlükə kəşfiyyatı (Threat Intelligence)

platformalarını və cavab alətlərini əhatə edir. Əsas funksiyalara

24/7 monitorinq, hadisənin aşkarlanması (Alerting), analiz (Triage),

hadisəyə reaksiya (Incident Response) və təhlükə ovçuluğu (Threat Hunting)

daxildir. SOC məqsədi təhlükəsizlik risklərini azaltmaq və şirkətin

kritik aktivlərini qorumaqdır.

23. SOC səviyyələri və rollar


SOC adətən üç əsas səviyyəyə (Tier) bölünür. Tier 1 (Triage Analyst)

ilkin siqnalları təhlil edir və yanlış həyəcanları ayırır.

Tier 2 (Incident Responder) təsdiqlənmiş insidentləri daha

dərindən araşdırır və aradan qaldırma tədbirlərini icra edir.

Tier 3 (Threat Hunter/Expert) mürəkkəb təhdidləri axtarır və yeni

müdafiə strategiyaları hazırlayır. Hər bir rol spesifik təcrübə tələb

edir və bütün bu rolların birgə işi təhlükəsizlik insidentlərinə

effektiv reaksiya verməyi təmin edir.

24. SOC alətləri və problemlər


SOC alətləri təhlükəsizlik fəaliyyətini dəstəkləyən proqram

təminatıdır. Əsas alətlər SIEM, SOAR (Security Orchestration,

Automation and Response) və TIP (Threat Intelligence Platform)

platformalarıdır. SOC nəsilləri texnologiyanın inkişafı ilə dəyişir

(məsələn, 1-ci nəsil sadə loq analizi, 3-cü nəsil isə avtomatlaşdırma və AI).

Əsas problemlər yüksək səviyyəli mütəxəssis çatışmazlığı, çoxlu

siqnal səsi (alert fatigue) və alətlərin bahalığıdır.

25. Autentifikasiya və faktorlar


Autentifikasiya istifadəçinin iddia etdiyi şəxs olduğunu

təsdiqləmə prosesidir. Üç əsas faktoru var: Bildiyiniz bir

şey (parol), Sahib olduğunuz bir şey (token, telefon) və

Siz olduğunuz bir şey (barmaq izi, üz tanıma). Çoxfaktorlu

Autentifikasiya (MFA) təhlükəsizliyi əhəmiyyətli dərəcədə

artırır. Reallaşdırma texnologiyalarına SAML, OAuth, Kerberos

və biometrik skanerlər daxildir.

26. Avtorizasiya modelləri


Avtorizasiya autentifikasiya edilmiş istifadəçinin hansı resurslara

və hansı səviyyədə çıxış etmək hüququnun olduğunu müəyyən etmə

prosesidir. Əsas modelləri DAC (Discretionary Access Control),

MAC (Mandatory Access Control) və ən geniş yayılmış RBAC

(Role-Based Access Control) təşkil edir. RBAC istifadəçilərə

rollar təyin edir və hər rolun müəyyən hüquqları olur.Protokollarına

RADIUS və TACACS+ daxildir.

27. NAS, RAID və TACACS+


NAS (Network Attached Storage) şəbəkəyə qoşulan və mərkəzləşdirilmiş

fayl yaddaşı xidməti təklif edən ixtisaslaşdırılmış serverdir.

İşləmə prinsipi fayl səviyyəli məlumatlara şəbəkə protokolları

(məsələn, SMB, NFS) vasitəsilə çıxışı təmin etməkdən ibarətdir.

RAID (Redundant Array of Independent Disks) NAS-da məlumatların

davamlılığını və performansını artırmaq üçün istifadə olunur.

TACACS+ (Terminal Access Controller Access-Control System Plus) isə AAA

(Autentifikasiya, Avtorizasiya, Mühasibat) xidməti təmin edən bir protokoldur.

28. DHCP server və snooping


DHCP Server (Dynamic Host Configuration Protocol) şəbəkədəki

cihazlara avtomatik olaraq IP ünvanı,alt şəbəkə maskası və

digər konfiqurasiya məlumatlarını təyin edir. DHCP Relay Agent

fərqli şəbəkə seqmentlərindəki müştərilərin DHCP serverlə əlaqə

qurmasına kömək edir. DHCP Snooping isə kommutatorlarda icazəsiz

DHCP serverlərdən gələn zərərli cavabları bloklayaraq şəbəkəni

DHCP hücumlarından qoruyan bir təhlükəsizlik xüsusiyyətidir.

29. NAT texnologiyası


Şəbəkə Ünvanının Tərcüməsi (NAT - Network Address Translation)

bir IP ünvanını digərinətərcümə edən bir marşrutlaşdırma

texnologiyasıdır. Əsas məqsədi şəxsi (private) IP ünvanlarını

(məsələn, 192.168.1.x) İnternetdə istifadə edilən ictimai (public)

IP ünvanlarına çevirməkdir.PAT/NAPT (Port Address Translation)

bir ictimai IP ünvanını minlərlə şəxsi ünvana xidmət etmək üçün

port nömrələri ilə birləşdirərək IPv4 ünvan çatışmazlığını yumşaldır.

30. Şəbəkə hücumları və müdafiə


Şəbəkə hücumları şəbəkənin məxfilik, bütövlük və əlçatanlıq

xüsusiyyətlərini pozmağa yönəlib. Əsas hücumlara DDoS

(Distributed Denial of Service), Malware (zərərli proqramlar),

Phishing və MiTM (Man-in-the-Middle) hücumları daxildir. Reallaşma

(müdafiə) texnologiyalarına Firewall-lar, IDS/IPS

(Intrusion Detection/Prevention Systems), VPN-lər

(Virtual Private Networks)və Endpoint Security həlləri daxildir.
